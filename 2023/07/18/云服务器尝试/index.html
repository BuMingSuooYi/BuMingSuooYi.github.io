下载docker

  1. 更新系统：

    更新 CentOS 服务器上的软件包和操作系统本身，确保系统有最新的补丁、安全修复和功能改进。

    执行 sudo yum update 命令会连接到 CentOS 软件源，检查更新，然后下载安装这些更新。

  2. 安装 Docker 依赖包：

    1. yum-utils：提供与 yum 软件包管理器相关的程序，用在 CentOS 上管理软件包。
    2. device-mapper-persistent-data：是device-mapper 存储驱动的持久数据包，用来提供 Docker 存储驱动的支持。
    3. lvm2：逻辑卷管理器 ，是 Linux 系统上的一个存储管理工具。Docker 使用 LVM 作为一种可选的存储后端，管理容器和镜像的存储。

  3. 添加 Docker 软件源：

    执行命令添加一个 “docker-ce.repo” 文件到 /etc/yum.repos.d/ 目录，该文件包含 Docker 的官方软件源的配置信息，指定了 Docker 的下载地址和相关信息，这样我们就可以通过 yum 包管理器来管理 Docker 软件包。

  4. 安装Docker社区版

    这个命令安装了 Docker CE（社区版）的核心组件，包括 Docker Engine（Docker 运行时环境）、Docker CLI（命令行界面）和 containerd（容器运行时）。

    安装过程中询问是否同意添加 Docker 官方 GPG 密钥时，同意就行，这是Docker 官方软件源的密钥，用来验证软件源的身份和完整性确保从官方源下载的软件包未被篡改或被替换为恶意软件。

    GPG（GNU Privacy Guard）密钥用于验证软件源的身份和完整性。通过验证软件源的密钥，您可以确保从官方源下载的软件包未被篡改或被替换为恶意软件。

  5. 启动Docker

    sudo systemctl start docker

  6. 设置Docker开机自启动

    sudo systemctl enable docker

    可以在服务器重启时，保持容器和应用程序的重新运行 ；在使用自动化工具或脚本进行服务器部署和配置，可以确保 Docker 服务在每次部署后都能自动启动，简化部署过程 。

  7. 查看Docker版本信息

    sudo docker version

默认位置：

  1. Docker 安装位置： Docker 的二进制文件、配置文件和其他相关文件通常位于 /usr/bin/docker 和 /etc/docker 目录下。这些文件用于安装和配置 Docker 引擎本身。

  2. Docker 镜像位置： Docker 镜像文件默认存储在 /var/lib/docker 目录下。在该目录中，Docker 会为每个镜像创建一个子目录，每个子目录的名称是镜像的唯一标识符。镜像的具体文件将存储在该子目录中。

  3. Docker 容器位置： Docker 容器的文件系统位于 /var/lib/docker 目录下的容器子目录中。每个容器都有一个唯一的标识符，Docker 将为每个容器创建一个子目录，该子目录包含容器的文件系统。

    注意：容器的文件系统并不直接可见，而是由 Docker 引擎管理和隔离的。您可以通过 Docker 提供的命令（如 docker cp）来访问容器内的文件。
