String 适用于不需要修改字符串内容的情况，由于不可变性，它具有更好的内存管理和安全性。

StringBuffer 适用于多线程环境下需要修改字符串的情况，因为它是线程安全的，但性能相对较低。

StringBuilder 适用于单线程环境下需要修改字符串的情况，因为它性能较好，但不具备线程安全性。

日期时间

  1. 过时的Data类和新的日期和时间 API

    java.util.Date 类在 Java 中已经过时，主要是因为它在设计上存在以下问题，无法很好地应对复杂的日期和时间操作，以及时区等问题。

    • 不可变性： java.util.Date 类在设计上是可变的，这导致了在多线程环境下的线程安全性问题。

    • 精度问题： Date 类精确到毫秒，但并没有提供处理纳秒级别精度的能力。

    • 时区问题： Date 类没有考虑时区的概念，这在跨时区应用中会导致问题。

    • 易混淆性：Date` 类中的一些方法名不够清晰，易混淆。

    Java 8 （既JDK1.8）引入了新的日期和时间 API，位于 java.time 包中，来解决这些问题。 java.time 包包含了以下类：

    • LocalDate：表示日期，无时区信息。
    • LocalTime：表示时间，无日期和时区信息。
    • LocalDateTime：表示日期和时间，无时区信息。
    • ZonedDateTime：表示带有时区的日期和时间。
    • Duration 和 Period：用于处理时间间隔和日期间隔。
    • Instant：表示时间线上的一个点，精确到纳秒。
    • 等等…

    所以我们以下的笔记只学习java.time 包中常用类
